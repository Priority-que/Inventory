# 核心业务流程说明

## 1. 文档目的

本文档用于描述系统核心业务流程，帮助开发人员、测试人员和项目成员统一理解业务流转方式和关键状态变化。

## 2. 主业务流程

系统的主流程如下：

采购申请 -> 审批 -> 采购订单 -> 供应商确认 -> 到货登记 -> 入库 -> 库存更新

## 3. 采购申请流程

### 3.1 流程说明
- 采购员创建采购申请
- 填写申请信息和采购明细
- 保存为草稿或直接提交审批
- 提交后进入待审批状态
- 审批通过后进入可生成采购订单状态
- 审批驳回后退回修改

### 3.2 状态流转
草稿 -> 待审批 -> 已通过 / 已驳回 -> 已转采购单

## 4. 审批流程

### 4.1 流程说明
- 采购主管查看待审批申请
- 审核申请内容
- 通过或驳回
- 记录审批意见

### 4.2 规则
- 只有具备审批权限的角色可以审批
- 审批结果必须记录日志
- 审批通过后，申请单不可再任意编辑

## 5. 采购订单流程

### 5.1 流程说明
- 根据已审批通过的采购申请生成采购订单
- 指定供应商、价格、交期等信息
- 订单生成后待供应商确认
- 供应商确认后进入执行阶段
- 根据到货情况更新订单执行状态

### 5.2 状态流转
待供应商确认 -> 已确认 -> 部分到货 -> 已完成 / 已关闭

## 6. 供应商协同流程

### 6.1 流程说明
- 供应商登录系统
- 查看待确认订单
- 确认接单
- 填写预计交期
- 反馈供货情况

### 6.2 规则
- 供应商仅能查看自身订单
- 未审核供应商不能参与正式采购

## 7. 到货与入库流程

### 7.1 流程说明
- 仓库根据采购订单执行到货登记
- 核对到货数量与订单信息
- 创建入库单
- 确认入库后自动更新库存

### 7.2 规则
- 支持分批到货、分批入库
- 入库数量不能超过剩余可入库数量
- 入库成功后形成库存流水

## 8. 库存更新流程

### 8.1 流程说明
- 入库确认后系统更新库存数量
- 同时记录库存流水
- 如果低于安全库存则触发预警

### 8.2 规则
- 库存必须通过业务单据驱动变更
- 不允许无来源直接修改库存台账

## 9. 异常场景

### 9.1 采购申请被驳回
- 状态变为已驳回
- 采购员可修改后重新提交

### 9.2 供应商延期
- 订单保留执行中状态
- 可记录延期说明
- 后续可触发提醒或标记异常

### 9.3 到货数量不足
- 系统记录部分到货
- 剩余部分继续等待后续到货

### 9.4 入库失败
- 不更新库存
- 保留原始到货记录
- 由仓库岗重新处理
