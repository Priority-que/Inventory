# 角色与权限设计

## 1. 文档目的

本文档用于明确系统中各类角色的职责边界、可访问菜单和核心操作权限，为权限控制、菜单设计和接口鉴权提供依据。

## 2. 角色说明

### 2.1 系统管理员
职责：
- 维护系统基础数据
- 管理用户、角色、菜单
- 审核供应商
- 查看系统日志和基础统计数据

### 2.2 采购岗
职责：
- 创建采购申请
- 提交审批
- 生成采购订单
- 跟踪订单执行情况

权限细分：
- 采购员：创建申请、维护订单
- 采购主管：审批采购申请

### 2.3 仓库岗
职责：
- 处理到货登记
- 执行入库
- 处理退货
- 查看库存台账和预警信息

### 2.4 供应商
职责：
- 维护自身资料
- 查看采购订单
- 确认接单
- 反馈交期和供货状态

## 4. 权限设计原则

- 菜单权限与接口权限统一按角色控制
- 供应商只能查看与自身相关的数据
- 审批权限不得下放给普通采购员
- 入库相关操作仅限仓库岗执行
- 管理员拥有系统配置和审计查看权限，但不直接参与业务执行
