## 3. 测试目标

验证采购入库主链路是否可以按角色完整流转：

采购员提交采购申请 -> 采购主管审批通过 -> 采购员基于已通过申请生成采购订单 -> 供应商确认采购订单 -> 供应商发货或线下发货状态衔接 -> 仓库岗做到货登记 -> 仓库岗做入库确认 -> 系统库存增加。

重点确认：

* 角色菜单和按钮是否符合职责。
* 单据状态是否按流程正确变化。
* 采购申请、采购订单、到货登记、入库单之间是否正确关联。
* 入库确认后库存数量是否按采购数量增加。
* 页面展示、接口返回、数据变化是否一致。

## 4. 测试范围

本次包含：

* 登录与角色菜单检查。
* 采购申请页面：新增申请、维护申请明细、提交申请、查看状态。
* 采购审批页面：审批通过、查看审批后状态。
* 采购订单页面：基于已通过申请生成采购订单，核对供应商、物料、数量、金额。
* 供应商订单确认页面：供应商只查看自己的订单并确认。
* 仓库到货管理页面：基于已确认采购订单进行到货登记。
* 仓库入库管理页面：基于到货登记生成入库单并确认入库。
* 库存变化核对：入库前后库存数量对比。

本次不包含：

* Agent 模块测试。
* 代码修复、页面重构、数据库结构调整。
* 权限认证、JWT、登录逻辑修改。
* 供应商物料关系扩展设计。
