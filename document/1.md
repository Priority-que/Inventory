# 系统功能总览

## 1. 文档目的

本文档用于从整体上说明“面向中小制造企业的供应商协同采购与仓储履约平台”的系统定位、角色划分、模块组成和核心业务闭环，帮助项目成员快速理解系统全貌。

## 2. 系统定位

本系统面向中小制造企业采购与仓储履约场景，围绕采购执行全过程进行数字化管理，重点解决以下问题：

- 采购申请、审批、下单流程线下化，流转效率低
- 供应商资料和合作状态缺乏统一管理
- 采购订单执行进度不透明，交付异常无法及时发现
- 到货、入库、库存记录不统一，账实容易不一致
- 采购、仓库、管理层之间缺少统一的数据视图

系统的核心业务主链路为：

采购申请 -> 审批 -> 采购订单 -> 供应商确认 -> 到货登记 -> 入库处理 -> 库存更新

## 3. 系统角色

系统共设置 4 类核心角色：

- 系统管理员
- 采购岗
- 仓库岗
- 供应商

其中采购岗在权限上可细分为：
- 采购员
- 采购主管

## 4. 系统模块

系统功能按业务划分为以下模块：

- 认证与登录模块
- 系统管理模块
- 供应商管理模块
- 采购管理模块
- 仓储管理模块
- 消息通知模块
- 统计报表模块

## 5. 核心功能概述

### 5.1 认证与登录模块
- 用户登录
- JWT 鉴权
- 角色权限控制
- 当前登录用户信息获取

### 5.2 系统管理模块
- 用户管理
- 角色管理
- 菜单管理
- 字典管理
- 操作日志
- 基础配置

### 5.3 供应商管理模块
- 供应商档案维护
- 资质文件上传
- 供应商审核
- 供应商状态管理

### 5.4 采购管理模块
- 采购申请管理
- 采购审批管理
- 采购订单管理
- 订单进度跟踪

### 5.5 仓储管理模块
- 到货登记
- 入库管理
- 退货处理
- 库存台账
- 库存预警

### 5.6 消息通知模块
- 审批提醒
- 到货提醒
- 库存预警提醒
- 系统消息

### 5.7 统计报表模块
- 采购金额统计
- 供应商采购分析
- 入库趋势分析
- 库存预警统计

## 6. 一期建设范围

一期建议实现以下最小可用闭环：

- 登录与权限
- 供应商管理
- 采购申请
- 审批流
- 采购订单
- 到货登记
- 入库管理
- 库存查询
- 操作日志
- 基础统计报表

## 7. 二期扩展方向

- 对账与付款跟踪
- 供应商评级
- 导入导出
- 多仓库管理
- 更复杂审批流
- 仪表盘看板
