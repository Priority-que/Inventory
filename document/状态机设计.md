# 核心状态机设计

## 1. 文档目的

本文档用于定义系统核心业务对象的状态、状态流转规则和触发条件，避免开发过程中出现状态混乱、业务边界不清晰的问题。

## 2. 状态机设计原则

- 每个核心业务对象必须有明确状态。
- 状态只能按定义好的规则流转，不允许任意跳转。
- 状态变更必须由明确业务动作触发。
- 关键状态变更必须记录操作日志。
- 页面按钮展示和接口调用都应基于当前状态控制。

## 3. 核心业务对象

本系统一期重点定义以下状态机：

- 供应商状态机
- 采购申请状态机
- 审批记录状态机
- 采购订单状态机
- 到货登记状态机
- 入库单状态机
- 库存预警状态机

## 4. 供应商状态机

### 4.1 状态定义

- 待提交
- 待审核
- 审核通过
- 审核驳回
- 合作中
- 已停用

### 4.2 状态说明

- 待提交：供应商资料未完善，尚未提交审核。
- 待审核：已提交审核，等待管理员处理。
- 审核通过：已通过审核，可参与采购合作。
- 审核驳回：审核未通过，需修改资料后重新提交。
- 合作中：已正式参与业务合作。
- 已停用：禁止继续参与新采购业务。

### 4.3 状态流转

待提交 -> 待审核  
待审核 -> 审核通过  
待审核 -> 审核驳回  
审核驳回 -> 待提交  
审核通过 -> 合作中  
合作中 -> 已停用

### 4.4 关键规则

- 只有审核通过或合作中的供应商可被采购单选择。
- 已停用供应商不能新增采购订单。
- 审核驳回后必须修改资料后重新提交。

## 5. 采购申请状态机

### 5.1 状态定义

- 草稿
- 待审批
- 已通过
- 已驳回
- 已撤回
- 已转采购单

### 5.2 状态说明

- 草稿：采购员新建但尚未提交审批。
- 待审批：已提交，等待采购主管审批。
- 已通过：审批通过，可生成采购订单。
- 已驳回：审批未通过，需修改后重新提交。
- 已撤回：申请人在审批前主动撤回。
- 已转采购单：已根据申请生成采购订单。

### 5.3 状态流转

草稿 -> 待审批  
待审批 -> 已通过  
待审批 -> 已驳回  
待审批 -> 已撤回  
已驳回 -> 草稿  
已通过 -> 已转采购单

### 5.4 关键规则

- 草稿状态允许编辑和删除。
- 待审批状态不可直接编辑采购明细。
- 已通过状态才能生成采购订单。
- 一张申请单可根据实际设计决定是否支持拆单。
- 如果一期想简单些，可以先设计为一张申请单对应一张采购订单。

## 6. 审批记录状态机

### 6.1 状态定义

- 待处理
- 已通过
- 已驳回

### 6.2 状态流转

待处理 -> 已通过  
待处理 -> 已驳回

### 6.3 关键规则

- 同一条审批记录只能处理一次。
- 审批动作必须记录审批人、审批时间、审批意见。
- 审批结果会同步驱动采购申请状态变化。

## 7. 采购订单状态机

### 7.1 状态定义

- 待确认
- 已确认
- 执行中
- 部分到货
- 已完成
- 已关闭
- 已取消

### 7.2 状态说明

- 待确认：采购员创建订单后，等待供应商确认。
- 已确认：供应商已确认接单。
- 执行中：订单进入履约阶段，尚未有到货完成结果。
- 部分到货：已有部分到货或部分入库。
- 已完成：订单全部完成到货和入库。
- 已关闭：因业务结束主动关闭。
- 已取消：订单作废，不再执行。

### 7.3 状态流转

待确认 -> 已确认  
已确认 -> 执行中  
执行中 -> 部分到货  
部分到货 -> 已完成  
执行中 -> 已关闭  
部分到货 -> 已关闭  
待确认 -> 已取消

### 7.4 关键规则

- 待确认状态下供应商可确认订单。
- 已取消订单不允许继续到货和入库。
- 已完成订单表示全部履约结束。
- 部分到货表示仍有未完成履约明细。

## 8. 到货登记状态机

### 8.1 状态定义

- 待登记
- 已登记
- 部分到货
- 全部到货
- 异常到货

### 8.2 状态说明

- 待登记：订单尚未发生到货登记。
- 已登记：至少发生一次正常到货登记。
- 部分到货：到货数量小于订单应到数量。
- 全部到货：累计到货数量等于订单应到数量。
- 异常到货：存在短交、超交、破损、不合格等情况。

### 8.3 状态流转

待登记 -> 已登记  
已登记 -> 部分到货  
部分到货 -> 全部到货  
已登记 -> 异常到货  
部分到货 -> 异常到货

### 8.4 关键规则

- 到货登记由仓库岗执行。
- 累计到货数量不得超过订单允许范围。
- 异常到货应保留异常说明和处理记录。

## 9. 入库单状态机

### 9.1 状态定义

- 待入库
- 已入库
- 已取消
- 部分入库异常

### 9.2 状态说明

- 待入库：已创建入库单，尚未确认入库。
- 已入库：入库成功，库存已更新。
- 已取消：入库单作废。
- 部分入库异常：存在数量异常或处理未完成。

### 9.3 状态流转

待入库 -> 已入库  
待入库 -> 已取消  
待入库 -> 部分入库异常  
部分入库异常 -> 已入库

### 9.4 关键规则

- 只有确认入库后才能更新库存。
- 已入库状态不可直接修改数量。
- 入库数量不得超过可入库数量。

## 10. 库存预警状态机

### 10.1 状态定义

- 正常
- 低库存预警
- 超储预警
- 已处理

### 10.2 状态流转

正常 -> 低库存预警  
正常 -> 超储预警  
低库存预警 -> 已处理  
超储预警 -> 已处理  
已处理 -> 正常

### 10.3 关键规则

- 库存变化后自动判断是否触发预警。
- 已处理仅表示人工确认，不代表库存已恢复。
- 当库存恢复到合理范围时，状态回到正常。

## 11. 状态与按钮控制关系

### 11.1 采购申请
- 草稿：显示编辑、删除、提交审批
- 待审批：显示查看、撤回
- 已驳回：显示编辑、重新提交
- 已通过：显示生成采购订单
- 已转采购单：仅查看

### 11.2 采购订单
- 待确认：显示供应商确认、取消订单
- 已确认：显示查看执行进度
- 执行中：显示到货跟踪
- 部分到货：显示继续到货、继续入库
- 已完成：仅查看
- 已关闭：仅查看

### 11.3 入库单
- 待入库：显示确认入库、取消
- 已入库：仅查看
- 异常状态：显示异常处理

## 12. 开发建议

- 所有状态字段统一使用枚举管理。
- 状态流转逻辑放在 Service 层处理。
- 不同状态对应的接口权限和按钮显隐要统一。
- 状态变更必须记录日志。
- 一期先保证主链路状态稳定，不要过早引入过多复杂状态。
